MAA Task Bar Hero 使用说明

项目名称：
MAA Task Bar Hero

作者说明：
Concept & direction by Marcus Xu
Built with Codex

一、软件用途

MAA Task Bar Hero 是一个用于 Steam 游戏 TaskBarHero 的视觉自动化辅助工具。
它不会读取游戏内存，也不会修改游戏文件。
它通过截图识别游戏画面，再根据识别结果自动选择刷图路线、检测 Boss 提示、识别宝箱掉落，并按路线循环执行。

二、使用前准备

1. 打开 Steam 并启动 TaskBarHero。
2. 确保游戏窗口标题中包含 TaskBarHero。
3. 游戏内需要打开并保持以下界面：
   - 英雄/背包界面
   - 传送门/地图界面
   - 日志窗口
4. 尽量不要最小化游戏窗口。
5. 建议保持游戏窗口大小和初始化时一致。

三、启动软件
如果你使用的是发布版：
1. 打开文件夹：
   dist 
2. 然后打开文件夹：
   MAA Task Bar Hero
3. 双击：
   MAA Task Bar Hero.exe

如果你使用的是源码版：
1. 安装依赖：
   pip install -r requirements.txt
2. 运行：
   python route_planner_gui.py

四、路线选择

软件打开后，左侧是可用刷图目标，右侧是当前刷图循环路线。

操作方式：

1. 在左侧选择一个关卡。
2. 点击“添加 →”加入右侧路线。
3. 可使用“上移”“下移”调整刷图顺序。
4. 可使用“← 移除”删除右侧选中的路线。
5. 可使用“清空路线”清空当前路线。
6. 点击“保存路线”会保存到 farm_plan.json。

右侧路线就是 Bot 实际循环执行的顺序。

五、启动 Bot

设置好路线后，点击：
保存并启动 Bot
软件会自动保存 farm_plan.json，并启动自动化 Bot。
下方“Bot 运行日志”会显示当前执行过程，例如：
- 是否找到游戏窗口
- 当前路线
- 是否切换难度
- 是否切换章节
- 是否找到关卡
- 是否检测到 Boss 提示
- 是否检测到蓝色宝箱
- 是否推进到下一条路线

六、停止 Bot

点击：
停止 Bot
即可停止当前运行的 Bot。
如果关闭软件时 Bot 还在运行，软件会提示是否停止 Bot 并关闭窗口。

七、后台输入模式说明

界面中有一个选项：
全自动后台挂机（实验版）
默认建议不要勾选。
不勾选时：
Bot 会使用真实鼠标点击游戏窗口，稳定性更好。
勾选时：
Bot 会尝试使用 Windows 后台消息点击，不移动真实鼠标。
这样理论上用户可以继续使用鼠标做别的事。
但注意：
TaskBarHero 不一定接受后台输入。
如果勾选后游戏没有反应，请取消勾选，使用默认模式。

八、蓝色宝箱逻辑

Bot 的主要判断逻辑是：
1. 等待 Boss WARNING 提示。
2. Boss 出现后，开始检测蓝色掉落。
3. 同时检测蓝色宝箱和蓝色日志文字。
4. 确认蓝色掉落后，自动打开蓝色宝箱。
5. 推进到下一条路线。
6. 自动切换到下一关卡。
7. 等待下一次 Boss。

如果启动时已经存在蓝色宝箱，Bot 会进入“孤立蓝箱恢复逻辑”：
如果蓝箱持续存在一段时间，或同时检测到蓝色日志，会认为这是已经发生的掉落，并继续推进路线。

九、路线导航逻辑

Bot 会自动完成：
1. 选择难度
2. 选择章节
3. 查找目标关卡
4. 查找关卡左侧圆点
5. 如果是白点，则点击进入
6. 如果是绿点，则认为当前已经在该关卡，无需重复点击
7. 进入等待 Boss 的状态

十、常见问题

1. 软件找不到游戏窗口
请确认 TaskBarHero 已经打开，并且窗口标题中包含 TaskBarHero。

2. Bot 没有点击游戏
请确认没有勾选“全自动后台挂机（实验版）”。
如果后台模式无效，请使用默认前台鼠标模式。

3. Bot 找不到关卡
可能原因：
- 游戏窗口大小变化太大
- 地图没有打开
- 目标关卡模板截图不准确
- 缺少 templates/levels 中对应的关卡图片

4. 某些关卡显示
说明该关卡暂时缺少必要模板，或者当前版本还没有完全支持。
需要补充对应截图模板后才能启用。

5. 想添加新关卡
通常需要添加：
- templates/levels/level_x_x.png

如果是新难度，还需要添加：
- templates/difficulty/anchor_difficulty_xxx.png
- templates/difficulty/tab_difficulty_xxx.png

然后在 route_config.py 中加入对应路线配置。

十一、文件说明

主要文件：
route_planner_gui.py
路线选择和启动界面。

main.py
Bot 主逻辑。

capture.py
截图游戏窗口。

detector.py
OpenCV 模板匹配和颜色检测。

route_config.py
关卡、路线、模板路径配置。

farm_plan.json
当前保存的刷图路线。

templates/
所有识别用图片模板。

debug_screenshots/
调试截图保存目录。

十二、注意事项

本工具基于屏幕识别，不是内存修改工具。
如果游戏 UI、窗口大小、语言、字体或画面位置发生很大变化，识别可能失败。

建议：
- 保持游戏窗口大小稳定
- 保持地图和日志窗口打开
- 遇到问题时查看 Bot 运行日志
- 必要时保存截图并更新模板
